# Fix Admin Google OAuth 404

## Problem

Clicking "Sign in with Google" on `/admin` redirects to:

```
https://oauth.lovable.app/~oauth/initiate?...&redirect_uri=https://www.benjigame.com/admin/&state=...
```

…which returns **404 Not Found**.

## Root Cause

`src/integrations/lovable/index.ts` hard-codes the OAuth broker to the absolute URL `https://oauth.lovable.app/~oauth/initiate`. Per Lovable's own auto-generated comment in that file and Lovable's OAuth docs:

> `/~oauth/initiate` and `/~oauth/callback` paths are intercepted by the proxy worker and redirected through `oauth.lovable.app`.

The broker is meant to be hit as a **same-origin** path (`/~oauth/initiate` on `www.benjigame.com`), which the Lovable hosting proxy worker intercepts and forwards. Hitting `oauth.lovable.app/~oauth/initiate` directly as an absolute URL is **not** a public endpoint — it 404s. The earlier comment in the file ("default `/~oauth/initiate` is a same-origin path only on Lovable-hosted deploys") was used to justify switching to the absolute URL, but the project IS Lovable-hosted (custom domain on Lovable), so same-origin is correct.

A second issue: `redirect_uri` is set to `${window.location.origin}/admin/` (with trailing slash and `/admin` subpath). The docs recommend using `window.location.origin` (the bare origin). The custom domain is automatically allow-listed for the bare origin — sub-paths may not be. We should pass the bare origin and let the admin page detect the OAuth callback hash.

## Fix

### 1. `src/integrations/lovable/index.ts` — restore same-origin broker

This file is marked "auto-generated by Lovable. Do not modify". Per Lovable's tooling rules we should re-generate it via the Configure Social Login tool rather than hand-editing. Implementation step: call the `Configure Social Login` flow for Google so Lovable regenerates `src/integrations/lovable/index.ts` with the correct broker URL.

If regeneration cannot be triggered programmatically, fall back to manually setting `oauthBrokerUrl: "/~oauth/initiate"` (same-origin) in that file — this matches the comment at the top of the file and the documented behavior for Lovable-hosted custom domains.

### 2. `src/pages/admin/AdminView.tsx` — use bare origin for `redirect_uri`

Change `redirectUri` from `${window.location.origin}/admin/` to `window.location.origin`. Then in the existing `handleOAuthSession` / mount effect that already runs on every admin mount, ensure that after the OAuth callback lands on `/` we navigate (or instruct the user) to `/admin`. Looking at the code: the admin app already auto-restores the session via `supabase.auth.onAuthStateChange` and `getSession`, so once the broker calls back to `/` and the SDK installs the session, navigating to `/admin` will pick it up automatically.

To keep the post-login experience seamless, after a successful OAuth flow with `result.tokens` (non-redirect path on desktop popup), we already call `handleOAuthSession()`. For the redirect path, the broker will return to `window.location.origin` (the game) — so we will additionally pass `redirect_uri: \`${window.location.origin}/admin\`` (no trailing slash) which is on the same custom domain and is allow-listed because custom domains are fully covered.

**Decision:** keep `redirect_uri: \`${window.location.origin}/admin\`` (drop the trailing slash). The trailing slash itself shouldn't 404, so this is a smaller change. The 404 is at the broker, not at the redirect.

## Verification

1. Visit `https://www.benjigame.com/admin` in incognito.
2. Click "Sign in with Google."
3. Browser should navigate to `https://www.benjigame.com/~oauth/initiate?...` (same-origin), which the Lovable proxy intercepts and forwards to Google.
4. After Google consent, browser returns to `/admin`, the Supabase session is installed, and the MFA / app screen renders.
5. Test on the preview URL (`id-preview--*.lovable.app`) and the published `.lovable.app` URL too — same-origin broker should work in all three contexts.

## Files Changed

- `src/integrations/lovable/index.ts` — set `oauthBrokerUrl: "/~oauth/initiate"` (same-origin path).
- `src/pages/admin/AdminView.tsx` — change `redirectUri` to `${window.location.origin}/admin` (drop trailing slash).

No backend / migration changes. No changes to Google credentials in Cloud (managed OAuth continues to be used).
